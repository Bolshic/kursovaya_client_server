Курсовая работа: «Управление студенческим общежитием»

	Описание проекта

Клиент-серверное приложение для автоматизации работы администрации общежития.  
Позволяет вести учёт студентов, групп, факультетов, комнат, корпусов; управлять заселением, выселением и переселением; формировать отчёты; искать студентов по ФИО и паспорту; изменять данные.

	Технологии

- **Backend**: C# / .NET 10, ASP.NET Core Web API, Dapper, Npgsql, Swagger
- **База данных**: PostgreSQL
- **Frontend**: HTML5, CSS3, JavaScript (ES6), Bootstrap 5

	Требования для запуска

- [.NET 10 SDK](https://dotnet.microsoft.com/download)
- [PostgreSQL](https://www.postgresql.org/download/) (версия 14 или выше)
- Любой современный браузер (Chrome, Firefox, Edge)

	Установка и запуск

1. Создание базы данных

Откройте терминал (или pgAdmin) и выполните:

```sql
CREATE DATABASE dormitory;

2. Выполнение SQL-скриптов
В папке DatabaseScripts находятся два файла. Выполните их в указанном порядке (например, через psql или pgAdmin):

01_CreateDatabase.sql – создаёт все таблицы, триггеры, функции, хранимые процедуры и представления.

02_FillTestData.sql – очищает таблицы, сбрасывает последовательности и заполняет базу тестовыми данными (факультеты, корпуса, группы, студенты, комнаты, проживание, квитанции, платежи).

Пример выполнения через psql:
psql -U ваш_пользователь -d dormitory -f DatabaseScripts/01_CreateDatabase.sql
psql -U ваш_пользователь -d dormitory -f DatabaseScripts/02_FillTestData.sql

3. Настройка строки подключения
Откройте файл DormitoryAPI/appsettings.json.
Измените строку DefaultConnection под свои параметры PostgreSQL:
{
  "ConnectionStrings": {
    "DefaultConnection": "Host=localhost;Database=dormitory;Username=ваше_имя;Password=ваш_пароль"
  }
}

4. Запуск бэкенда (API)
Перейдите в папку DormitoryAPI и выполните:

dotnet restore
dotnet run

API запустится на http://localhost:5183.
Документация Swagger доступна по адресу: http://localhost:5183/swagger

5. Открытие клиентской части
Дважды кликните на файл index.html или откройте его в браузере.
Убедитесь, что API запущен, иначе данные не будут загружаться.

	Возможности системы
Вкладка «Студенты и фильтры»
Фильтрация студентов по факультету, курсу, группе.

	Поиск по ФИО.
Отображение списка студентов с указанием комнаты и корпуса (если проживают).

	Вкладка «Отчёты»
Проживающие – список студентов, проживающих в общежитии в данный момент.
Свободные комнаты – комнаты со свободными местами.
Должники – студенты с неоплаченными квитанциями.
Статистика по корпусам – заполняемость комнат и мест.

	Вкладка «Действия»
Поиск студента по ФИО с отображением паспорта, группы, комнаты и корпуса.
Заселение – выбор корпуса, свободной комнаты и даты заселения.
Выселение – указание даты выселения для проживающего студента.
Изменение данных – смена ФИО, группы, а также переселение в другую комнату (автоматическое выселение+заселение).

Возможные проблемы и их решение

Ошибка подключения к БД – проверьте правильность строки подключения в appsettings.json и запущен ли PostgreSQL.
Порт 5183 уже используется – измените порт в Program.cs и в index.html (переменная API_BASE).
CORS ошибка – в Program.cs уже настроена политика AllowAll, проблем быть не должно.
Не отображаются корпуса или комнаты – убедитесь, что в таблицах Building и Room есть данные (скрипт 02_FillTestData.sql их добавляет).
